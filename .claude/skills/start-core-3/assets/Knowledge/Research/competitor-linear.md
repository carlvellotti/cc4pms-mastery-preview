# Competitive Profile: Linear

## Overview

Linear is a modern project management tool built specifically for software teams. Founded in 2019, it has gained rapid adoption among engineering-led startups and mid-market companies by prioritizing speed, keyboard-first UX, and opinionated workflows. Often described as "what Jira should have been."

## Enterprise Tier: Linear Enterprise (Plus + custom)

### Pricing
- **Plus plan:** $8/user/month (annual) — includes most features, this is where most teams operate
- **Enterprise plan:** Custom pricing, typically $12-16/user/month for larger orgs
- No free tier for teams (individual use only)
- Transparent pricing with minimal discounting — Linear generally doesn't negotiate heavily

### Key Enterprise Features
- SAML SSO (Plus plan and above)
- SCIM provisioning (Enterprise only)
- Audit log (Enterprise only, limited — activity-level, not field-level)
- Advanced roles and permissions
- Org-level settings and workspace management
- GitHub, GitLab, and Bitbucket integrations (deep — PR-to-issue linking, auto-close)
- Cycles (opinionated sprint model)
- Triage queue and intake workflows
- API-first design (GraphQL)
- SLA commitments available on Enterprise
- No native recurring tasks — workaround via API or third-party automations (Zapier)

## Enterprise Positioning
"Built for the way modern software teams work." Linear's pitch is anti-complexity. They explicitly position against Jira's bloat and Monday.com's lack of technical depth. Their messaging is developer-first: fast, opinionated, beautiful. Enterprise positioning is emerging but secondary — Linear leads with product quality and lets word-of-mouth drive adoption upward into larger orgs.

## Key Strengths
- **Speed:** The fastest UI in the category. Sub-100ms interactions, optimistic updates, feels native.
- **Developer love:** Consistently the highest-rated PM tool among engineers. Keyboard-first, no unnecessary clicks.
- **Opinionated workflows:** Cycles, triage, and project structure enforce good practices without configuration overhead.
- **Git integration depth:** PR linking, branch creation from issues, auto-status updates. Best-in-class for GitHub-native teams.
- **API quality:** GraphQL API is clean, well-documented, and genuinely API-first (their own UI uses it).
- **Design quality:** Consistent, polished UI that sets the standard for what modern PM tools should look like.

## Key Weaknesses
- **Enterprise feature gaps:** Limited audit logging, no data residency options, no IP allowlisting. Procurement teams flag these.
- **No recurring tasks:** Surprising gap for a tool this polished. Teams use Zapier or custom API scripts as workarounds, which frustrates non-technical users.
- **Cross-functional limitations:** Built for engineering teams. No portfolio/goals layer, limited reporting for leadership, no views designed for non-technical stakeholders.
- **Customization constraints:** Opinionated design means fewer configuration options. Teams that need custom workflows or fields feel constrained.
- **Automation:** Basic automations only. No visual automation builder. Power users hit limits quickly.
- **Scale concerns:** Adopted bottom-up by engineering teams, but untested at true enterprise scale (5,000+ users). Limited case studies from Fortune 500.

## What Enterprise Customers Say
- Engineers are passionate advocates — they describe Linear as "a joy to use" and resist switching to anything else.
- PMs appreciate the speed but wish for better reporting, roadmap views, and cross-team visibility.
- Enterprise buyers flag the lack of audit logs and compliance certifications as deal-blockers for regulated industries.
- Several teams we've spoken with use Linear for engineering and Asana or Notion for cross-functional work — a split that creates friction.
- The lack of recurring tasks comes up frequently. Teams say it's a surprising omission for a tool that otherwise nails the basics.
