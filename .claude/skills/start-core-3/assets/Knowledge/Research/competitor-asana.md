# Competitive Profile: Asana

## Overview

Asana is a work management platform originally built for team coordination and project tracking. Founded by Facebook co-founder Dustin Moskovitz, it has strong roots in non-technical teams but has been aggressively expanding into enterprise and engineering workflows over the past two years.

## Enterprise Tier: Asana Enterprise+

### Pricing
- **Enterprise plan:** $13.49/user/month (annual, custom pricing for 500+ seats)
- **Enterprise+ plan:** Custom pricing, typically $18-22/user/month for advanced security and admin features
- Free trial available for Business tier; Enterprise requires sales contact
- Discounts available for annual commitments and nonprofit/education orgs

### Key Enterprise Features
- SAML SSO with custom authentication policies
- SCIM provisioning and de-provisioning
- Admin audit log API
- Custom branding and workspace theming
- Advanced reporting with cross-project dashboards (Portfolios)
- Data loss prevention (DLP) integrations
- Service accounts for automated workflows
- Approval workflows and forms
- Recurring tasks (native — set any task to repeat on a schedule)
- 99.9% SLA (Enterprise+)
- Data export API

## Enterprise Positioning
"Work management for the whole organization." Asana's enterprise pitch centers on being the platform that bridges engineering, marketing, operations, and leadership. They emphasize cross-functional visibility — everyone from IC to CEO in one tool. Their go-to-market is increasingly top-down, with CIO/COO-level conversations about "operational excellence."

## Key Strengths
- **UX quality:** Consistently rated as the most intuitive project management tool. Onboarding is fast.
- **Cross-functional appeal:** Works well for marketing, ops, HR, and engineering in the same org.
- **Portfolios and Goals:** Strong strategic layer — connect daily tasks to company OKRs.
- **Recurring tasks:** Native, flexible recurring task support. Users can set daily, weekly, monthly, or custom recurrence patterns with automatic assignment.
- **Automation builder:** Visual, no-code automation (Rules) that non-technical users can configure.
- **Customer success:** Highly responsive. Enterprise customers get a dedicated CSM and implementation support.

## Key Weaknesses
- **Engineering workflow gaps:** No native sprint planning, story points, or backlog management. Engineers find it too "lightweight."
- **API limitations:** REST API is functional but not API-first. Rate limits are restrictive for heavy integrations.
- **Custom fields:** Less flexible than Jira or TaskFlow. No computed/formula fields.
- **Reporting depth:** Dashboards look good but lack the analytical depth data teams need (no pivot tables, limited filtering).
- **Performance at scale:** UI slows noticeably with 10,000+ tasks in a project. Large portfolios can lag.

## What Enterprise Customers Say
- Non-technical teams love it — it's the tool they actually open every day without being forced to.
- Engineering teams tolerate it but often maintain a shadow workflow in GitHub Issues or Linear.
- Managers praise the Portfolios view for executive reporting and status updates.
- Power users hit ceiling quickly — they want more custom fields, better formulas, and deeper API access.
- Some enterprise buyers concerned about the gap between "work management" messaging and actual technical project management capability.
