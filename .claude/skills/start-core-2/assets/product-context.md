# TaskFlow Product Context

## What We Build
TaskFlow is a project management tool for mid-market teams (50-500 employees). We sit between the heavy enterprise tools (Jira) and the lighter consumer-grade ones (Asana, Monday.com). Our value prop: powerful enough for engineering-led organizations, clean enough that non-technical teammates actually use it.

## Market Position
- ~120 paying teams, 85% retention
- Primary buyer: engineering managers and product leaders
- Primary users: PMs, engineers, designers, occasionally execs
- Pricing: Starter ($12/user/mo), Growth ($25/user/mo), Enterprise (custom)
- Key differentiator: workflow automation engine — competitors either don't have it or gate it behind enterprise pricing

## Current Focus: v3.0 (Q2)
Four major features shipping this quarter:
1. **Enterprise SSO** (SAML/OIDC) — biggest blocker for enterprise deals. 3 prospects waiting on this.
2. **Advanced Reporting Dashboard** — current reporting is basic. Enterprise customers need custom metrics, saved views, scheduled reports.
3. **Custom Workflow Builder** — visual editor for automation rules. Currently our workflows are template-based. This lets teams build their own.
4. **Bulk Operations API v2** — the v1 API can't handle operations on more than 100 items. Enterprise customers hit this constantly.

## Competitive Landscape
| Competitor | Strength | Weakness | Our Angle |
|-----------|----------|----------|-----------|
| Jira | Enterprise features, integrations | Heavy, slow, hated by non-devs | "Jira power without Jira pain" |
| Asana | Clean UX, easy onboarding | Shallow for technical teams | "Asana simplicity with real workflows" |
| Monday.com | Marketing/ops friendly | Not built for engineering | "Actually built for how product teams work" |
| Linear | Fast, dev-loved | No non-dev features | "Linear speed for the whole team" |
